# 比价方案租户端操作说明（优化版）

## 一、整体介绍

### 核心功能与操作流程

租户端菜单包含两大核心模块：「比价方案」和「报表中心」。 **操作步骤**：先创建完整的比价方案，再基于该方案发布报表（1个比价方案支持关联发布多个报表，灵活适配不同分析场景）。

### 核心逻辑

比价方案负责定义“比什么、用哪些数据比、按什么维度比、怎么计算结果”；报表中心负责将方案中定义的维度、指标，以透视表等形式可视化展示，支持拖拽配置行列维度和值维度。

## 二、相关概念

### 1. 比对模型

*   定义：平台端提前配置并启用的“基础模板”，是创建比价方案的前提，核心作用是明确4个关键信息：
    
    1.  方案可使用的数据集（包含比对数据集、基准数据集）；
        
    2.  不同数据集之间的关联规则（通过**比对目标**字段自动关联）；
        
    3.  比对目标（即“比什么”）；
        
    4.  支持的比对维度、查询条件范围。
        
*   关键规则：一旦选择比对模型创建方案后，该模型不可修改，需提前确认适配业务需求。
    

### 2. 数据集

*   定义：存储对比所需数据的集合，分为两类，二者配合实现“主辅对比”：
    
    *   比对数据集：核心数据集，是“被比对”的主数据来源，选完比对模型后自动确定，不可变更；
        
    *   基准数据集：辅助数据集，用于提供对比基准（如历史数据、其他渠道数据），一个比对模型可包含0个或多个基准数据集。
        
*   关联规则：所有数据集通过「比对目标字段」自动关联（比对目标是n个字段的集合，不同数据集保证存在相同编码的字段，因此可精准关联）。
    

### 3. 比对目标

*   定义：明确“方案到底比什么”，是多个字段的组合（比如“物料编码+物料名称”，即方案核心对比“物料”）。
    

### 4. 比对对象

*   定义：基于「比对数据集」的数据配置，相当于Excel中的“主表”，是方案的核心配置项
    
*   核心配置项（3个关键维度）：
    
    1.  比对维度：定义方案可从哪些角度分析对比结果（比如比对目标是物料时，维度可设为供应商、子公司、采购模式、采购日期等）；
        
        *   来源：维度来自平台端模型数据集的已有字段，租户端只能选择启用哪些，不能自定义新增；
            
        *   作用：后续报表中可将这些维度拖拽为“行/列维度”，实现多维度透视分析。
            
    2.  比对范围：定义比对数据的筛选条件（比如“仅对比集团渠道的物料价格”“采购时间范围=2023年1月-12月”）；
        
        *   关键规则：这些查询条件是固化的，后续在报表中无法修改（仅只读）；若需调整筛选条件，必须返回比价方案中编辑修改，不支持在报表中动态调整。
            
    3.  比对指标：定义该数据集可生成的计算结果（相当于透视表中的“值”），支持自定义公式，满足最大值、最小值、平均值、求和等场景。
        
        *   指标类型：系统会根据公式自动判断是“聚合指标”还是“明细指标”，无需手动选择；
            
        *   配置方式：
            
            *   复制预设指标：平台端已预先定义数据集中的常用指标，租户端可直接复制，且支持修改公式；
                
            *   自定义指标：从零创建，无需额外选择数据集（因比对对象已关联比对数据集），直接从数据集中选择字段，搭配运算符/函数配置公式（比如“含税价×折扣率=折后价”）。
                

### 5. 基准对象

*   定义：用于提供对比基准的数据配置，相当于Excel中的“辅表”，**可选配置**（不配置则报表仅展示比对对象数据）。
    
*   数据来源：可选择「比对数据集」或「基准数据集」；
    
    *   灵活配置：允许与比对对象使用同一数据集，但设置不同的筛选条件（比如比对对象筛选“2023年数据”，基准对象筛选“2022年数据”，实现同比分析）。
        
*   核心配置项：
    
    1.  基准范围：对基准数据的筛选条件（逻辑同“比对范围”，固化不可在报表中修改）；
        
    2.  基准指标：定义基准数据的计算规则（配置逻辑、支持的公式 / 函数同比对指标），**同样分为聚合指标和明细指标两类**，由系统根据公式自动判断（聚合 = 分组计算汇总值，明细 = 保留原始数据字段值）；
        

### 6. 计算指标

*   定义：与比对指标、基准指标的“单数据集内计算”不同，计算指标支持**跨多个数据集**的指标联动计算，核心用于生成“差异类”指标（比如差异额、差异率）。
    
*   示例：若比对指标为“2023年采购价”，基准指标为“2022年采购价”，可配置计算指标“差异额=2023年采购价-2022年采购价”“差异率=（2023年采购价-2022年采购价）/2022年采购价”。
    
*   支持功能：可使用+、-、\*、/等运算符，以及MAX（最大值）、MIN（最小值）、AVG（平均值）、SUM（求和）等预设函数。
    

### 7. 聚合与明细（重要）

*   聚合：对数据先按维度分组，再计算一个汇总值（比如按“物料+子公司”分组，计算该组合下的最低价）；
    
*   明细：不对数据分组，直接保留原始数据的字段值，最终结果行数和原始数据一致（比如每条采购记录的“含税价”）；
    
*   对应指标：聚合指标（按分组计算）、明细指标（保留原始数据），由系统根据指标公式自动判断。

### 2. 对象关联规则

方案中配置的维度、指标，最终需在报表中使用，关联规则如下：

*   比对指标的维度来源：比对**目标字段 + 报表中启用的比对维度**；
    
    *   聚合类比对指标：按“比对目标 + 启用的比对维度”分组计算结果；
        
    *   明细类比对指标：维度值直接取自原始数据行的对应字段。
        
*   基准指标的维度来源：
    

*   聚合类基准指标：**仅比对目标中定义的字段（固定不可修改）**，同一比对目标组合仅显示 1 条汇总数据；
    
*   明细类基准指标：维度值取自原始数据行的对应字段（与明细类比对指标一致），结果行数与原始数据一致；
    
*   示例：若比对目标是 “物料编码 + 物料名称”，聚合类基准指标按该组合显示 1 条数据，明细类基准指标会显示该组合下所有原始数据对应的指标值。
    

## 四、指标使用详细示例

### 1. 明细指标示例

*   定义：不分组，保留原始数据的字段值，无需聚合计算。
    
*   示例指标：含税价、不含税价（均为明细指标）；
    
*   界面展示： ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/Lk3lbmbdw1WwpOm9/img/eda80b4c-1dc2-48e9-afb8-f061dd244799.png)![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/Lk3lbmbdw1WwpOm9/img/5fb77430-c601-489e-8ef8-cd95be50ce89.png)
    
*   展示效果：结果行数和原始数据完全一致，能看到每条采购记录的具体含税价、不含税价，无汇总计算。
    

### 2. 聚合指标示例

*   定义：按维度分组计算，需配置聚合公式（如最大值、最小值）。
    
*   示例场景：配置2个聚合指标“最高价”（公式：MAX(含税价)）、“最低价”（公式：MIN(含税价)）；
    
*   配置步骤（还原截图操作）：
    
    1.  新增比对指标，名称分别设为“最高价”“最低价”；
        
    2.  选择关联字段“含税价”，分别配置公式为MAX(含税价)、MIN(含税价)； 界面配置截图： ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/Lk3lbmbdw1WwpOm9/img/6cbe9c58-bff5-47b8-8421-2284fba865c5.png)
        
    3.  在报表中选择启用的维度（示例：物料+供应商+采购日期）； 界面配置截图： ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/Lk3lbmbdw1WwpOm9/img/6af431fe-417d-442e-a516-672a71af3db6.png)
        
*   展示结果：
    
    1.  当维度选得细（物料+供应商+采购日期）时，结果类似明细（因每个组合仅1条原始数据，聚合后仍为1条）； 截图效果： ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/Lk3lbmbdw1WwpOm9/img/9a98596f-79f1-45d5-9191-617a6236d2df.png)
        
    2.  当维度选得粗（仅“物料”）时，同一物料仅显示1条聚合结果（1个最高价、1个最低价）； 截图效果： ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/Lk3lbmbdw1WwpOm9/img/9de69e9a-148b-40f0-93bd-3144c5bde09b.png)
        
*   **关键结论：聚合指标的结果行数，由“启用的维度粗细”决定，维度越粗，结果越精简。**
    

### 3. 明细指标和聚合指标同时使用的问题

*   核心问题：会触发“笛卡尔积”（排列组合）现象，导致数据冗余、不易分析；
    
*   单一组合示例：
    
    *   某维度下，比对对象明细指标有5条结果，比对对象聚合指标有1条结果；
        
    *   最终该维度会显示 **5（明细）×1（聚合）=5 条数据**（明细指标每条数据都与聚合指标数据关联）。
        
*   多对象组合示例（更复杂）： 若同时启用“比对对象”“基准对象A”“基准对象B”的指标，最终某维度下的结果行数 = 比对对象明细指标数 × 比对对象聚合指标数 × 基准对象A明细指标数 × 基准对象A聚合指标数 × 基准对象B明细指标数 × 基准对象B聚合指标数；
    
    *   备注：若某类指标未配置（比如基准对象B无明细指标），则无需乘以该类数量。
        
*   建议：尽量不同时启用多个对象的明细指标，优先单独使用明细指标或聚合指标，避免报表数据混乱。
    

## 五、比价方案列表页面

### 核心功能与字段说明

列表页面用于集中管理所有已创建的比价方案，支持查看、操作方案及关联报表，核心信息与操作如下：

#### 1. 列表核心字段

| 字段 | 说明 |
| --- | --- |
| 状态 | 展示方案当前状态（启用/停用），决定方案是否可用于发布新报表 |
| 管理组织 | 方案所属组织，按组织权限过滤展示 |
| 方案编码 | 系统自动生成（租户内唯一），无需用户干预 |
| 方案名称 | 自定义标识名称（如“2023年物料采购价vs集团基准价比对”） |
| 报表数量 | 关联已发布的报表总数，点击可进入报表管理子页面 |
| 更新人 | 最后一次修改方案的用户 |
| 更新时间 | 最后一次修改方案的时间 |

#### 2. 核心操作按钮（含权限与前提限制）

| 按钮 | 功能说明 | 操作限制 |
| --- | --- | --- |
| 启用 | 将“停用”状态的方案切换为“启用”，启用后可发布新报表 | 仅对“停用”状态的方案可见 |
| 停用 | 将“启用”状态的方案切换为“停用” | 停用后不影响已发布报表的正常查看和使用 |
| 发布报表 | 进入“报表配置页面”，基于当前方案创建新报表 |  |
| 编辑 | 修改方案的基础信息、比对范围、指标配置等（比对模型不可修改） | 无状态限制，修改后不影响已发布报表的历史配置 |
| 删除 | 彻底删除当前比价方案 | 前提：需先删除该方案关联的所有报表，否则无法删除 |
| 日志 | 查看方案的创建、修改、启用/停用等操作记录 | 所有状态的方案均支持查看 |

#### 3. 报表管理子页面（点击“报表数量”进入）

该页面以卡片形式展示当前方案关联的所有报表，核心信息与操作如下：

*   报表卡片展示内容：报表名称、报表描述、关联方案名称（即当前方案）；
    
*   操作按钮（按用户权限动态显示）：查看报表（查看最终数据结果）、查看报表配置（查看布局/查询条件等配置）、编辑报表（修改报表配置）、删除报表（移除该报表，不影响方案）、查看报表操作日志（记录报表的创建/修改/删除记录）。
    

> 示例：若方案“2023年物料采购价对比”关联了3张报表，点击“报表数量=3”，会看到3张报表卡片，每张卡片都显示“报表名称：物料-子公司采购价明细”“报表描述：查看各子公司物料采购明细价与基准价差异”“关联方案：2023年物料采购价对比”，且显示当前用户有权限的操作按钮（如普通用户仅显示“查看报表”，管理员显示全部按钮）。

## 六、报表配置页面（发布报表核心步骤）

报表配置页面用于将已创建的比价方案，通过拖拽布局、设置查询条件等操作，生成可视化报表，页面布局与操作逻辑如下：

### 1. 页面整体布局

页面分为三大区域，功能分工明确： ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/3M0OzeZEV4dKoqze/img/7f60e70b-504b-447e-8ddd-d518e0c37504.png)

*   左侧（1/8宽度）：报表可用元素区，展示比价方案中配置的所有维度和指标；
    
*   右侧上部：查询条件区（默认收起），可设置数据筛选条件；
    
*   右侧中部：报表布局配置区，支持拖拽元素到行/列/指标维度；
    
*   右侧下部：报表预览区，展示配置后的报表样式（非实时数据结果）。
    

### 2. 基本信息配置（报表基础标识）

| 字段 | 配置说明 | 示例 |
| --- | --- | --- |
| 报表名称 | 自定义报表名称，需清晰标识报表用途 | “2023Q4物料采购价-供应商对比报表”“子公司物料价与集团基准价差异表” |
| 方案名称 | 固定文本，自动带出关联的比价方案名称，不可编辑 | 若关联方案为“2023年物料采购价对比”，则此处直接显示该名称 |
| 报表分组 | 用于分类管理报表，支持选择系统预设“默认”分组或自定义分组 | 自定义分组示例：“2023年度报表”“供应商对比类报表”“子公司分析类报表” |
| 报表描述 | 填写报表用途、适用场景等信息，方便他人理解 | “用途：分析2023Q4各供应商的物料采购价与集团基准价的差异；适用场景：采购议价参考” |

### 3. 报表可用元素区（左侧）

展示比价方案中配置的所有可用于报表的元素，无需额外添加，直接拖拽使用：

*   比对维度：如供应商、子公司、采购月份、入库月份、采购模式等；
    
*   比对指标：如含税价（明细）、不含税价（明细）、平均采购价（聚合）等；
    
*   基准指标：如集团基准价（聚合）、2022年历史价（明细）等（分聚合/明细类型，同比对指标逻辑）；
    
*   计算指标：如差异额（比对价-基准价）、差异率（(比对价-基准价)/基准价）等。
    

> 注意：元素会展示比价方案中配置的全部内容，不支持搜索筛选，需直接查找拖拽。

### 4. 查询条件区（右侧上部）

用于设置数据筛选条件，分为三大块，支持精准过滤报表数据： ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/3M0OzeZEV4dKoqze/img/f5852878-47e4-48f3-a060-d9b99fbd8271.png)

**（1）查询条件分类与配置**

| 分类 | 配置说明 | 示例 |
| --- | --- | --- |
| 比对对象 | 筛选比对数据集的数据，条件来自比对模型中数据集的预设查询条件 | 选择“供应商类型”，操作符“=”，值“核心供应商”；选择“入库月份”，操作符“≥”，值“2023-10” |
| 基准对象 | 按具体基准对象独立配置（如“集团价”“集团历史价”各有独立条件），条件来自比对模型 | 给“集团价”设置“采购渠道=集团直采”；给“集团历史价”设置“采购时间≤2022-12” |
| 指标 | 按指标设置数值范围筛选，支持操作符（<、≤、≥、>），条件为区间或单一阈值 | 选择“含税价”，操作符“介于”，开始值“100”，结束值“500”；选择“差异率”，操作符“>”，值“10%” |

**（2）配置操作与规则**

*   添加条件：点击“添加”按钮，在弹窗中选择需要的查询条件类型、操作符和值； ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/3M0OzeZEV4dKoqze/img/e9cc4c69-4e6a-4679-a717-b3cdf000fde7.png)
    
*   条件生效规则：配置的查询条件仅对报表中使用的对象/指标生效（如未使用“A指标”，则A指标的筛选条件无效）；
    
*   只读条件规则：比价方案中“比对范围/基准范围”已使用的查询条件，在报表中会显示为灰色不可编辑，且不可重复配置。
    

### 5. 数据过滤逻辑（关键规则）

查询条件的过滤分为两步，先后顺序影响最终结果：

1.  数据集过滤：比对对象/基准对象的查询条件生效，过滤原始数据集（影响指标计算，如筛选后再算最高价/平均价）；
    
    *   示例：方案中比对范围设“供应商=A”，报表中加查询条件“入库月份=2023-10”，则指标计算仅用“A供应商2023年10月的原始数据”；
        
2.  结果过滤：指标的查询条件生效，过滤已计算完成的报表结果（不改变指标值，仅隐藏不满足条件的行）；
    
    *   示例：计算出差异率后，设置“差异率>10%”，报表仅显示差异率超10%的行，未显示行的差异率计算结果不变。
        

### 6. 报表布局配置（右侧中部）

按标准透视表逻辑配置，支持拖拽元素调整布局，核心规则如下：

**（1）布局维度配置**

| 布局维度 | 支持的元素来源 | 配置规则与示例 |
| --- | --- | --- |
| 行维度 | 比对对象（固定）+ 比对维度 | 比对对象固定在最前面，不可修改顺序；示例：行维度=物料编码→物料名称→供应商→子公司 |
| 列维度 | 仅比对维度 | 示例：按“采购月份”分列，查看不同月份的价格对比；按“子公司”分列，对比各子公司采购价 |
| 指标 | 比对指标、基准指标、计算指标 | 不可重复添加；计算指标需依赖的比对/基准指标已启用（系统自动校验），如“差异率”需先添加“比对价”和“基准价” |

**（2）配置操作示例**

1.  拖拽“物料编码”“物料名称”到行维度（比对对象固定在前）；
    
2.  拖拽“采购月份”到列维度；
    
3.  拖拽“含税价”（比对指标）、“集团基准价”（基准指标）、“差异率”（计算指标）到指标区；
    
4.  预览区会显示：按“物料编码+物料名称”分行、“采购月份”分列，每个单元格展示对应含税价、基准价和差异率的报表样式。
    

### 7. 指标配置

针对指标的补充配置，满足“查看指标来源属性”的需求（如最高价来自哪个供应商），配置步骤与规则如下：

**（1）弹窗打开与基础配置**

*   打开方式：点击指标区的目标指标，选择“指标配置”打开弹窗；
    
*   默认样式（未开启分组展示）：
    
    *   默认分组名称：报表级共享名称，修改后所有未单独设置分组名称的指标同步生效；
        
    *   分组展示：默认关闭，开启后可配置指标独立属性。
        

**（2）开启分组展示后的配置**

*   弹窗样式： ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/3M0OzeZEV4dKoqze/img/44011cae-c707-4f9b-b08f-0b1eb3f129aa.png)
    
*   关键配置项：
    
    1.  分组名称：指标独立名称，不与其他指标共享（如给“最高价”设分组名称“最高采购价信息”）；
        
    2.  指标属性：仅MAX（最高价）、MIN（最低价）公式的指标支持配置（AVG等公式配置后保存会报错），可选择需要展示的来源属性（如供应商名称、采购组织）；
        
    3.  展示模式：
        
        *   明细模式：满足条件的记录逐条展示，每条记录的属性占一行；
            
            *   示例：物料A的最高价来自3个供应商，明细模式显示3行，每行分别对应1个供应商+最高价；
                
        *   汇总模式：同一分组与列维度下，指标值合并为一行，属性值去重后用逗号拼接；
            
            *   示例：物料A的最高价来自3个供应商，汇总模式显示1行，供应商列显示“供应商1,供应商2,供应商3”+ 统一最高价。
                

**（3）指标分组规则**

1.  指标属性永远显示在对应指标的前面；
    
2.  只要有1个指标启用分组展示，报表会自动添加“指标分组”列维度，用于区分属性和指标的归属；
    
3.  指标分组的顺序与指标在报表中的顺序一致；
    
4.  同名称的指标分组会自动合并。
    

> 配置效果示例：给“最高价”启用分组展示，选属性“供应商名称”，分组名称“最高采购价”；报表会新增“指标分组”列，显示“最高采购价”，后续紧跟“供应商名称”和“最高价”列，清晰区分属性与指标。

### 8. 权限管理

**（1）默认权限规则**

报表发布后，仅创建人可查看和操作，其他用户无权限访问。

**（2）权限配置操作**

1.  点击页面右上方“权限管理”按钮，打开授权弹窗； ![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/3M0OzeZEV4dKoqze/img/dabaf8ca-1efa-4ad5-97d4-79123c678ef8.png)
    
2.  按组织粒度授予权限，权限级别分4级（包含关系，从低到高）：
    
    *   查看：仅能查看报表结果，无修改/删除权限；
        
    *   编辑：可查看报表、修改报表配置（布局、查询条件等）；
        
    *   删除：可查看、编辑、删除报表；
        
    *   管理：包含以上所有权限，且可修改报表的权限分配；
        
3.  多组织权限规则：用户若有多个组织的权限，对该报表的最终权限取所有组织中的最高级别；
    
    *   示例：用户在A组织有“查看”权限，在B组织有“编辑”权限，最终拥有“编辑”权限。

## 七、报表中心（报表集中管理与查看）

报表中心是已发布报表的最终管理入口，支持按分组查找、查看报表结果，核心功能如下：

### 1. 页面布局与核心功能

![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/3M0OzeZEV4dKoqze/img/5780301a-d10f-43e1-8588-78fdd27add3a.png)

*   左侧：报表分组导航栏，展示所有报表分组（系统预设“默认”分组+用户自定义分组），支持展开/折叠分组；
    
*   右侧：报表展示区，根据左侧操作动态切换：
    
    1.  点击分组：右侧显示该分组下所有报表的卡片（与比价方案列表页“报表数量”子页面的卡片一致）；
        
    2.  点击报表：右侧显示该报表的实时数据结果（非预览样式），支持刷新数据、导出报表等操作。
        

### 2. 报表分组管理

*   分组创建：用户可手动新建自定义分组（如“2024Q1报表”“采购议价参考报表”），也可使用系统预设“默认”分组；
    
*   分组用途：用于分类管理报表，避免报表过多难以查找；
    
*   卡片展示信息：报表名称、报表描述、关联方案名称、操作按钮（按用户权限显示）。
    

### 3. 报表查看与操作

*   查看报表：点击左侧报表名称，右侧加载实时计算的报表结果，支持按查询条件筛选（若报表配置了查询条件）；
    
*   导出报表：部分版本支持导出Excel格式，方便离线分析；
    
*   其他操作：根据用户权限，还可进行编辑报表、删除报表、修改权限等操作（与报表配置页、报表管理子页面的操作一致）。
